# 樾哥厨房微信小程序 Demo

一个用于家里点菜和食材采购确认的微信小程序原生 demo。当前版本包含菜单分类、菜品食材、采购勾选、下单记录，以及基于微信云开发数据库的远程同步预留。

## 功能

- 菜单页：宝宝推荐、中餐、西餐分类，外卖/餐厅点餐式布局。
- 菜品信息：每道菜包含制作时间、标签、简介和所需食材。
- 采购确认：自动汇总已点菜品的食材，可勾选确认是否齐全。
- 远程下单：配置微信云开发后，A 用户提交订单，B 用户打开订单页可实时同步看到。
- 离线 demo：未配置云开发时，会使用本地缓存，方便先在手机预览交互。

## 本地运行

1. 打开微信开发者工具。
2. 选择“导入项目”，目录选择本仓库。
3. AppID 可先选择测试号或填你的微信小程序 AppID。
4. 编译后可在模拟器预览，也可以点“预览”扫码在手机使用。

## 开启远程同步

需要你提供或完成：

- 微信小程序 AppID。
- 在微信开发者工具中开通“云开发”。
- 创建云环境后，把环境 ID 填到 `utils/config.js` 的 `CLOUD_ENV_ID`。
- 在云数据库中新建集合：`family_orders`。
- 给集合设置合适权限。demo 阶段可用“所有用户可读，仅创建者可写”；家庭共用时建议后续加登录身份和家庭邀请码。

配置完成后，两台手机使用同一个小程序体验版或开发版，A 在“采购”页远程下单，B 在“订单”页会通过数据库监听同步刷新。

## 后续可替换内容

- 菜品数据在 `data/dishes.js`。
- 现在菜品图先用色块占位；后续可以把真实菜品图片放入 `assets/`，再在菜品数据里增加图片路径。
- 如需正式发布，还建议补充用户登录、家庭成员绑定、订单状态流转、采购缺货提醒和菜品后台管理。
